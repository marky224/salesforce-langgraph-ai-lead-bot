fix: improve discovery, confirmation, and Salesforce lead capture

- Discovery: TARS no longer asks what impact does that have when the
  visitor has already described the impact - it only follows up when a
  pain point is stated vaguely.

- Confirmation: summary now reads naturally first, with Does that all
  look correct as the last sentence instead of the opening line.

- Confirmation: removed internal fields (decision-maker status, company
  size tier, current solution) from the user-facing summary - only
  contact info and conversational context are shown.

- Description leak fix: transcript summary (Salesforce Description field)
  was being streamed to the frontend because it was generated inside the
  confirmation node. Moved it to the scoring node, which is never streamed.

- Company capture fix: extraction prompt now rejects vague descriptions
  (e.g. a 400-employee company) as company names. Added a deterministic
  routing guard that redirects back to lead capture if the visitor provided
  their name but no company name, preventing the LLM router from skipping
  to confirmation prematurely.

- Tests updated to reflect new confirmation node behavior (one LLM call)
  and new scoring node behavior (generates transcript summary).
